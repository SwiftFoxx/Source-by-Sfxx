# Naming

## Swift Types
- Types: `PascalCase`
- Properties/Functions: `camelCase`
- Booleans: `is/has/should` prefixes

## Files
- Match file names to primary type names.
- Prefer `FeatureNameView.swift` for views.

## Git
- Actions are verb‑first (`fetch`, `push`, `commit`).
- Status types should be descriptive (`StatusSummary`, `FileChange`).
